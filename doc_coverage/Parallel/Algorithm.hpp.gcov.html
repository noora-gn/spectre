      70             :  * 1. A Parallel::Algorithms::Singleton where there is only one
      71             :  * in the entire execution of the program.
      72             :  * 2. A Parallel::Algorithms::Array which holds zero or more
      73             :  * elements each of which is a distributed object on some core. An array can
      74             :  * grow and shrink in size dynamically if need be and can also be bound to
      75             :  * another array. That is, the bound array has the same number of elements as

      78             :  * one element per core and they are not able to be moved around between cores.
      79             :  * These are typically useful for gathering data from array elements on their
      80             :  * core, and then processing or reducing it.
      81             :  * 4. A Parallel::Algorithms::Nodegroup, which is similar to a
      82             :  * group except that there is one element per node. For Charm++ SMP (shared
      83             :  * memory parallelism) builds a node corresponds to the usual definition of a
      84             :  * node on a supercomputer. However, for non-SMP builds nodes and cores are
      85             :  * equivalent. An important difference between groups and nodegroups is that
      86             :  * entry methods (remote calls to functions) are not threadsafe on nodegroups.
      87             :  * It is up to the person writing the Actions that will be executed on the
      88             :  * Nodegroup Algorithm to ensure they are threadsafe.
      89             :  *
      90             :  * ### What is an Algorithm?
      91             :  * An Algorithm is a distributed object, a Charm++ chare, that repeatedly
